# Reports API

## Endpoints

### GET /reports/summary
Генерирует сводный отчет в формате CSV.

**Описание:**
- Строки: различные модели оборудования
- Столбцы: характеристики (динамически определяются из базы данных)
- Значения: все различные значения характеристики для модели с указанием контракта в скобках

**Пример ответа:**
Файл CSV с заголовками и данными в формате:
```
Model,Мощность,Частота,Вес
Система ультразвуковая A,"100W (договор-123); 150W (договор-456)","50Hz (договор-123)","5kg (договор-123, договор-456)"
Система ультразвуковая B,"200W (договор-789)","60Hz (договор-789)","7kg (договор-789)"
```

**Возвращает:**
- Content-Type: `text/csv; charset=utf-8`
- Имя файла: `summary_report_YYYY-MM-DDTHH-mm-ss.csv`

### GET /reports/stats
Возвращает статистику по данным в системе.

**Пример ответа:**
```json
{
  "success": true,
  "data": {
    "totalModels": 42,
    "totalCharacteristics": 1250,
    "totalContracts": 150
  }
}
```

## Формат данных в CSV

**Структура:**
- Первая колонка: название модели
- Остальные колонки: характеристики (динамически)
- Значения ячеек: `"значение1 (контракт1, контракт2); значение2 (контракт3)"`

**Особенности:**
- Множественные значения разделяются точкой с запятой
- Контракты указываются в скобках через запятую
- Пустые ячейки для отсутствующих характеристик
- CSV экранирование для специальных символов
